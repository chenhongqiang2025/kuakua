## ✍️ 日常工作流 · 发一篇新笔记

### 在电脑上（推荐）

1. 打开 **<https://chenhongqiang2025.github.io/kuakua/admin.html>**
2. 已经登录 → 直接进入；没登录 → 粘 Token → Verify
3. 左侧栏点 **`+ New post`**
4. 填写：
   - **Title** 中文标题
   - **English title**（可选 —— 想做双语就填）
   - **Date** 默认是现在
   - **Tags** 标签，Enter 或逗号分隔（如 `affirmation, mirror`）
   - **Cover image** 封面图（可选）—— 拖一张图进去自动上传
   - **Excerpt** 一句话简介（Journal 卡片上显示）
   - **Body** 正文，支持 Markdown
5. 点编辑器顶部 **`Preview`** 看渲染效果
6. 满意 → 点绿色的 **`Publish`** 按钮
7. **等约 1 分钟** GitHub Pages 重建 → 刷新 <https://chenhongqiang2025.github.io/kuakua/#journal> 看新文章

### 在 iPhone 上（路上有灵感）

1. 主屏幕点 **KUAKUA Admin** 图标（之前装的 PWA）
2. 流程同上 —— 写完点 Publish
3. iOS 键盘工具栏（v1.2 计划加）暂时还没做，长段落手机打字会稍慢
4. **小技巧**：iPhone 上文字用语音输入更快（键盘左下角的麦克风按钮）

### 编辑已发布的文章

1. Admin 左侧栏点列表里的任意一条
2. 改内容 → 状态栏会显示 `● Unsaved changes`
3. 点 **Publish** 覆盖更新

### 删除文章

1. Admin 选中一篇 → 编辑器右上角 **Delete** 按钮
2. 弹窗确认 → 文章从网站下线
3. ⚠️ 注意：文章 JSON 在 git 历史里仍然存在；上传过的图片也保留（节省以后恢复）

## 🔄 Token 过期了怎么办（约 5 分钟）

如果 Admin 弹出登录页 + 提示 "Token rejected"：

### 1. 删除旧 token

打开 <https://github.com/settings/personal-access-tokens> → 找到 `KUAKUA admin` → 点进去 → 拉到底 → **Revoke**

### 2. 新建 token（**和上次一模一样的设置**）

打开 <https://github.com/settings/personal-access-tokens/new>，照下面填：

| 字段 | 值 |
|---|---|
| Token name | `KUAKUA admin` |
| Description | `Admin CMS for KUAKUA brand site` |
| Resource owner | `chenhongqiang2025` |
| Expiration | **Custom** → 一年后的日期 |
| Repository access | **`Only select repositories`** → 选 `chenhongqiang2025/kuakua` |
| **Repository permissions → Contents** | **`Read and write`** ⚠️ 关键 |
| 其他所有 permissions | 全部 `No access` |

点 **Generate token** → 复制（关页面前一定复制！）

### 3. 在 Admin 替换

1. 打开 Admin → 应该自动回到登录页
2. 粘 Token → Verify
3. 完成

### 4. 重新设下一年的日历提醒

新过期日是今天 +1 年 → 在日历里挪一下提醒事件。

## 🆘 紧急 / 故障对照表

| 现象 | 原因 | 解决 |
|---|---|---|
| 发布后网站没变化 | GitHub Pages 重建中 | 等 1 分钟，硬刷新（Ctrl+F5 / Cmd+Shift+R） |
| 发布报 `Publish denied — your token can read but not write` | Token 权限错了 | 按上面"Token 过期了怎么办"重做，注意 Contents 要 Read **and** write |
| 登录报 `Token verified, but it lacks write permission` | 同上 | 同上 |
| 登录报 `Token rejected (401)` | Token 复制不完整 或 已 Revoke | 重新复制或新建 |
| 登录报 `repo isn't visible to it (404)` | Token 没选对仓库 | 重做 token，Repository access 选 `Only select repositories` 并选 `kuakua` |
| 看不到自己发的笔记 | 浏览器缓存 | 硬刷新（Ctrl+F5）；iOS Safari 在设置里清缓存 |
| Admin 看到 "Token rejected" 但 token 没过期 | localStorage 被清了 | 重新粘 Token 进去就行 |
| 不小心删错文章了 | 文章 JSON 还在 git 历史里 | 找 [Richard / Claude](https://github.com/chenhongqiang2025/kuakua/commits/main) 从 commit 历史恢复 |
| 想看是不是真的部署成功 | 看 Pages 部署状态 | <https://github.com/chenhongqiang2025/kuakua/deployments> 看最新一条是 ✓ Success |
